02 / 后端分层设计

五层架构，各司其职

Controller入口层
负责接收 HTTP 请求。 这里声明接口路径和请求方式，例如 @PostMapping；用 pojo 请求对象接收前端 JSON；调用 Service，并统一返回 Result<T>。
Service业务抽象
负责定义业务能力。 例如错题本需要新增、查询、删除、清空；Controller 面向 Service 接口调用，降低页面接口和具体实现的耦合。
ServiceImpl业务实现
负责处理具体规则。 这里会把 Request 转成实体对象，补充 userId、默认值、时间等字段，再调用 Mapper 完成数据库操作。
Mapper数据访问
负责执行 SQL。 使用 MyBatis 注解 SQL 读写 MySQL，例如把 WrongQuestion 写入 wrong_questions 表。
pojo数据载体
贯穿整条链路。 实体类对应数据库记录，Request 类接收前端提交，Summary 类承载统计结果，Result.java 统一包装接口返回。
03 / 基础设施支撑

数据库迁移与统一响应

Flyway 数据库版本管理

自动管理数据库 schema 版本，应用启动时自动执行 src/main/resources/db/migration/ 目录下的 SQL 脚本，确保所有环境数据库结构一致。

脚本命名规则：
V1__Create_study_notes_table.sql
V2__Create_wrong_questions_table.sql

Result.java 统一返回格式

所有接口返回都通过 Result.success() 或 Result.error() 包装，确保前端处理逻辑一致。

成功响应：{ code: 200, msg: "success", data: {...} }
失败响应：{ code: 400, msg: "错误信息", data: null }

06 / 典型链路：学习笔记

以保存学习笔记说明一次请求怎么走完

studyNote.js调用 POST /api/v1/study-notes，提交课程模块 ID 和笔记内容。
StudyNoteController用 @RequestBody StudyNoteSaveRequest 接收参数，进行参数校验后调用 Service。
StudyNoteService定义学习笔记业务接口，暴露 getByUserId 和 save 方法。
StudyNoteServiceImpl将请求对象转为 StudyNote 实体，补充 userId、created_at、updated_at 字段。
StudyNoteMapper执行 @Insert 注解 SQL，将记录写入 study_notes 表并返回自增 ID。
Result<StudyNote>返回完整的笔记对象，前端更新笔记列表并显示保存成功提示。
07 / AI 集成

AI 问答完整调用流程

配置加载从 application.properties 读取 dashscope.api-key、dashscope.model、dashscope.timeout 等配置。
AiConfig 配置类使用 @ConfigurationProperties 绑定配置，注入到 AiServiceImpl。
ai.js调用 POST /api/v1/ai/ask，提交用户问题、context_id 和模型参数。
AiController用 @RequestBody AiChatRequest 接收参数，校验问题长度和格式，调用 Service。
AiServiceImpl读取配置中的 API Key，组装对话历史，构造 ChatCompletion 请求。
DashScope SDK使用配置的 API Key 进行认证，调用通义千问大模型服务。
大模型响应返回流式或完整响应，包含 AI 回答内容、token 使用量、finish_reason。
响应包装提取回答内容，包装成 Result<AiResponse> 统一格式返回前端。
