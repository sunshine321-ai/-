01 · 研究背景

卷积核“看不见、摸不着”，
传统讲授容易停留在公式层面

↻

练习割裂

◎

反馈滞后

教师难以快速了解学习时长、薄弱知识点与模块使用情况。

因此，我希望把卷积核从“一个公式”变成可操作、可观察、可追踪的学习对象。

05 · 前后端调用关系

页面不直接访问数据库，所有数据统一经过 API

Vue ViewStudyPage.vue
API 模块studyNote.js
基础地址/api/v1
ControllerStudyNoteController
Service接口 → Impl
MapperStudyNoteMapper
MySQLstudy_notes

开发环境转发

Vite :5173 将 /api 和 /files 代理到 Spring Boot :8000。

请求模块化

页面只调用 src/api 中的函数，不在组件中散落后端地址和响应解析。

响应统一

后端返回 Result<T>，前端统一依据 success / data / msg 处理。

06 · 真实请求时序

以“自动保存学习笔记”为例

① 浏览器端

1
旧学习页产生 study-note-change 消息
2
StudyPage.vue 校验来源，并进行 500ms 防抖
3
调用 saveStudyNote(content)
4
发送 PUT /api/v1/notes JSON 请求
→

② Spring Boot

5
Controller 将 JSON 转为 StudyNoteSaveRequest 并校验
6
调用 StudyNoteService.save() 接口
7
StudyNoteServiceImpl 执行业务流程
8
Mapper 执行插入或更新 SQL
⇄

③ 数据与返回

9
MySQL 按 user_id 保存笔记
10
Mapper 查询最新记录并逐层返回
11
Controller 封装为 Result<StudyNote>
12
前端解析 JSON，页面保持无刷新保存
